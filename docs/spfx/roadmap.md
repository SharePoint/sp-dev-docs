# SharePoint Framework roadmap

The first release of the SharePoint Framework in February 2017 only contained support for client-side web parts. This was just a start on the journey to provide additional modern customization capabilities to SharePoint which we have provided since and we have an exciting upcoming roadmap with features across Microsoft 365. Please see the specific published features from the version release notes, listed in the table of content.

We will be updating this roadmap page and the new feature set schedule will be confirmed.

[!INCLUDE [sp-add-in-deprecation](../../includes/snippets/sp-add-in-spfx.md)]

Here's the current roadmap on the upcoming releases:

## Version 1.23 – May 2026 - Released on 13th of May

- Command set improvements for lists and libraries – grouping and potentially other improvements.
- Open-sourcing SPFx solution templates via GitHub and opening them for community contributions.
- Preview of a new open-sourced SPFx CLI for replacing the existing Yeoman generator – You will be able to introduce your company specific templates or adjustments on the scaffolded baseline for the SPFx solutions. We are decoupling the CLI itself from the SPFx release versions.
- Addressing npm audit reported security issues on the dependency tree.

## Version 1.23.1 – June 2026

- New and edit panel override in lists – support panel level override on the Microsoft Lists and SharePoint lists with SPFx.
- Addressing npm audit reported security issues on the dependency tree.

## Version 1.24 – June 2026

- Feature X to public preview (disclosed soon)
- Addressing npm audit reported security issues on the dependency tree.

## Version 1.25 – September 2026

- Feature X general availability (disclosed soon)
- Navigation customizers – Options to override navigation nodes and/or experiences with SPFx components.
- General availability of SPFx CLI.
- Addressing npm audit reported security issues on the dependency tree.

## Top of Mind

- React 18 support for SPFx solutions - This update is dependent on updating Microsoft provided web parts and experiences to use React 18 version. This work is in progress and hopefully we get an update out soon
