# Assess SharePoint modernization

Use the [Microsoft 365 Assessment tool](https://pnp.github.io/pnpassessment/index.html) for new SharePoint modernization assessments. The open-source tool collects data from your Microsoft 365 environment and generates reports that help you understand modernization, adoption, and retirement scenarios.

The available assessment modules include the **Classic SharePoint Pages Assessment**, which discovers classic wiki, web part, blog, and publishing pages, inventories their web parts, and measures their readiness to be transformed into modern pages.

## Get started

1. Review the [Microsoft 365 Assessment tool documentation](https://pnp.github.io/pnpassessment/index.html).
1. [Download the latest release](https://github.com/pnp/pnpassessment/releases).
1. [Configure authentication](https://pnp.github.io/pnpassessment/using-the-assessment-tool/setupauth.html).
1. [Run an assessment](https://pnp.github.io/pnpassessment/using-the-assessment-tool/assess.html).

For classic page modernization, see:

- [Classic SharePoint Pages Assessment](https://pnp.github.io/pnpassessment/classic/readme.html)
- [Run a classic pages assessment](https://pnp.github.io/pnpassessment/classic/assess.html)
- [Understand the generated report](https://pnp.github.io/pnpassessment/classic/report-intro.html)

> [!NOTE]
> The SharePoint Modernization Scanner is a legacy tool and is no longer actively maintained. Its documentation remains available under **Archive** for customers who need to interpret existing scanner output or continue an established scanner workflow.
